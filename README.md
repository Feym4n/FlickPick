## FlickPick

### 1. Название проекта

**FlickPick** — выбор фильма без споров

---

### 2. Цель проекта

Создать веб-приложение с интерфейсом в стиле Tinder, которое помогает компании друзей выбрать фильм для совместного просмотра.
Приложение автоматически формирует пул фильмов, предложенных участниками, и помогает группе прийти к единому выбору с помощью механики свайпов.

---

### 3. Основные пользователи

* **Создатель группы** — инициирует сессию выбора фильма, получает код для приглашения друзей.
* **Участники группы** — присоединяются по коду, добавляют свои фильмы и участвуют в голосовании.

---

### 4. Основные сценарии использования

#### 4.1. Создание группы

* Пользователь создает группу.
* Приложение генерирует уникальный 5-значный код (латинские заглавные буквы и цифры, например `A9T5F`).
* Создатель делится кодом с друзьями.

#### 4.2. Присоединение к группе

* Другие пользователи вводят код, чтобы присоединиться.
* После подключения им отображается список добавленных фильмов.

#### 4.3. Добавление фильмов

* Каждый участник может добавить 1–5 фильмов, которые он хочет посмотреть.
* Приложение подтягивает данные о фильме (постер, описание, рейтинг) через **Кинопоиск API**.

#### 4.4. Начало голосования

* Создатель группы нажимает кнопку **«Начать выбор»**.
* Всем участникам последовательно показываются карточки фильмов (свайп влево/вправо — дизлайк/лайк).

#### 4.5. Совпадения

* Когда все участники лайкнули один и тот же фильм, происходит **«матч»**.
* Приложение уведомляет всех, что найден фильм, который нравится всем.

| №  | Функция                    | Описание                                                   |
| -- | -------------------------- | ---------------------------------------------------------- |
| 1  | Создание и удаление группы | Создание группы с кодом доступа                            |

| 8  | Уведомления                | Информирование участников о результатах                    |
| 9  | Авторизация (опционально)  | Возможность входа по имени или соцсетям                    |
| 10 | Хранение данных            | Использование БД для хранения групп, пользователей, лайков |

### 6. Нефункциональные требования

* **Архитектура:** клиент-сервер
* **Фронтенд:** React / Next.js (адаптив под мобильные экраны)
* **Бэкенд:** Node.js / Express
* **База данных:** Firebase или PostgreSQL
* **Интеграции:** API Кинопоиска (постеры, описания, рейтинг)
* **UI/UX:** минимализм, свайпы как в Tinder, карточный интерфейс
* **Безопасность:** уникальные сессии по коду, токены пользователей
* **Производительность:** отклик интерфейса ≤ 200 мс

---

### 7. API-интеграция (пример запроса)

```js
GET https://kinopoiskapiunofficial.tech/api/v2.2/films?keyword=Inception
Headers: { "X-API-KEY": "<API_KEY>" }
```

Ответ содержит:

* Название фильма
* Постер (URL)
* Описание
* Год, рейтинг

---

### 8. Архитектура

```
Клиент (React/Next.js)
  ├── UI свайпов (карточки фильмов)
  ├── Экран группы (код, участники)
  └── API-клиент для сервера

Сервер (Node.js/Express)
  ├── REST API
  │    ├── /groups (создание, присоединение)
  │    ├── /films (добавление, получение)
  │    ├── /votes (лайки, результаты)
  └── WebSocket (реальное время)

БД (Firebase/SQL)
  ├── users
  ├── groups
  ├── films
  ├── votes
```

---

### 9. Поток пользователя (User Flow)

1. **Создать группу** → получить код → поделиться
2. **Присоединиться по коду**
3. **Добавить фильмы**
4. **Нажать “Начать выбор”**
5. **Свайпы всеми участниками**
6. **Матч → показ фильма-победителя**

---

### 10. MVP-функционал

* Создание / присоединение к группе
* Добавление фильмов с Кинопоиска
* Голосование свайпами
* Определение совпадения
